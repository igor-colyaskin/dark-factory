# BOOTSTRAP.md — Промпт для восстановления контекста

## Как использовать

Когда начинаешь новый чат (с Claude, или в Roo Code),
вставь текст ниже + приложи файлы CONCEPT.md и TRACKER.md.

## Промпт

---

Привет! Мы работаем над проектом Dark Factory (DF).
Это мой курсовой проект по Vibe Coding.

Ключевая информация:
- DF — веб-приложение, "фабрика приложений" с метафорой фастфуда
- Пользователь описывает приложение, DF автоматически строит его через AI-агентов
- Стек DF: Node.js + Express + Vanilla JS/CSS + OpenRouter API + JSON state
- 3 агента: Architect (Opus), Developer (Sonnet), Tester (Gemini Flash)
- Оркестратор — детерминированный код (state machine), НЕ AI
- Целевое приложение v0.1: простые Node.js + Express приложения
- Всё работает локально
- DF строится через Roo Code в VSCode (уровень 1)
- DF сама вызывает OpenRouter API напрямую (уровень 2)

Приложены файлы:
1. CONCEPT.md — полная концепция проекта
2. TRACKER.md — текущий статус и план шагов

Посмотри текущий статус в TRACKER.md и помоги мне
с [ОПИСАНИЕ ТЕКУЩЕЙ ЗАДАЧИ].

Обнови в нём последнюю строку — впиши конкретную задачу,
над которой работаешь. Например:

  "...помоги мне с реализацией server/orchestrator.js (шаг 2.2)"

  "...помоги мне с отладкой промпта для Architect (шаг 3.1)"

   вот ошибка: [текст ошибки]"
